## Decisions Made
- AppContext holds all 7 services plus engine, session_factory, settings as a dataclass for clean DI
- Tools access AppContext via `ctx.request_context.lifespan_context` (standard FastMCP lifespan pattern)
- All tools return `dict` via `model_dump(mode="json")` for MCP transport compatibility
- find_related_papers accepts `str | list[str]` and normalizes internally, deduplicating by highest score
- get_paper uses direct ORM `select(Paper)` query rather than SearchService for single-paper lookup (no ranking needed)
- Discovery module imported at bottom of server.py via noqa side-effect import for tool registration
