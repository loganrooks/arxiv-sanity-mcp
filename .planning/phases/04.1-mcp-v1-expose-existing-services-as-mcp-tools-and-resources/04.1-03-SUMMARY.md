## Decisions Made
- All tools use `source="mcp"` for audit trail provenance (distinguishes MCP-initiated actions from CLI/manual)
- `add_to_collection` auto-creates collection on ValueError (catches "not found" from add_papers, creates, retries)
- `create_watch` is a two-step operation: create saved query, then promote to watch (matches existing service separation)
- Paper resource manually serializes enrichment ORM fields (not a Pydantic model, unlike other services)
- Resources return `{"error": ...}` dicts on not-found (consistent with tool error pattern from Plan 02)
- Tool naming tests use `asyncio.run(mcp.list_tools())` for registration introspection (not function name checks)
