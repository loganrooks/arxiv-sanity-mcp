### Architectural Decisions (settled, from prior phases)
- **ADR-0001:** Exploration-first — multiple strategies coexist, no premature commitment
- **ADR-0002:** Metadata-first, lazy enrichment — ingest eagerly, enrich on demand
- **ADR-0003:** License and provenance first — track all provenance
- **ADR-0004:** MCP as workflow substrate — not a thin search wrapper

### Phase 04.1 Key Decisions
- Category Jaccard scored exactly once (in `score_category_overlap` only)
- `score_seed_relation` now author-only (was 60% category + 40% author)
- `score_profile_match` now 85% author + 15% query_boost (was 50% category + 35% author + 15% query)
- AppContext dataclass with 7 services for FastMCP lifespan DI
- Tools return dicts via `model_dump(mode='json')`
- `find_related_papers` accepts `str|list[str]` multi-seed, deduplicates by highest score, excludes seed IDs
- All MCP tools use `source="mcp"` for audit trail provenance
- `add_to_collection` auto-creates collection on ValueError (idempotent UX)
- `create_watch` is two-step: `create_saved_query` then `promote_to_watch`
- Paper resource is a composite view from 4 services (ORM + triage + enrichment + collections)
- Resources return `{"error": ...}` dicts on not-found

- Roadmap resequenced 2026-03-11: Phase 5 = MCP Validation (was Phase 6), Phase 6 = Content Normalization (was Phase 5)

- PREMCP fixes scoped into Phase 04.1 Plan 1 (not a separate quick task)
- Prompts deferred from Phase 04.1 to Phase 5 (design through practice, not speculation)
- Feedback loop (evaluation lenses, tension signals, calibration) is POST-Phase 5 work
