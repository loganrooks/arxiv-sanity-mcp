**literature_review.py** -- Guided multi-step discovery workflow:
```python
@mcp.prompt()
async def literature_review_session(
    seed_query: str,
    category: str | None = None,
    profile_slug: str | None = None,
    ctx: Context = None,
) -> list[UserMessage]:
```
Generates instructions that guide the agent through:
1. Search with seed_query (and category filter if provided)
2. Browse results, triage each paper (shortlist/dismiss/seen)
3. Find related papers from shortlisted seeds
4. Add keepers to a collection
5. Enrich top papers for citation/topic data
6. Repeat until diminishing returns

If profile_slug provided, include instruction to use profile-ranked search. Keep instructions concise (~300-400 tokens). List available tool names but do NOT include paper content or abstracts.

**daily_digest.py** -- Automated monitoring workflow:
```python
@mcp.prompt()
async def daily_digest(
    profile_slug: str | None = None,
    ctx: Context = None,
) -> list[UserMessage]:
```
Generates instructions to:
1. List all active watches (mention watch://{slug}/deltas resource)
2. For each watch with new papers: get_paper for each, quick triage assessment
3. Auto-shortlist high-confidence matches, flag borderline for human review
4. Summary of findings organized by watch/collection

If profile_slug provided, assess against profile. Concise (~250-350 tokens).

**triage_shortlist.py** -- Batch paper evaluation:
```python
@mcp.prompt()
async def triage_shortlist(
    collection_slug: str,
    profile_slug: str | None = None,
    ctx: Context = None,
) -> list[UserMessage]:
```
Fetches live collection state via AppContext to report paper count. Generates instructions to:
1. List papers in collection
2. For each: read paper resource (paper://{arxiv_id}), assess relevance
3. Recommend triage state with 1-2 sentence reasoning
4. Human confirms or overrides

Handle ValueError from show_collection (collection not found) by returning error message. Concise (~300-400 tokens).
