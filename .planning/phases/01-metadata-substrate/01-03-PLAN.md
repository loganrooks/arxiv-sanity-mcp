Search, browse, and discovery: fielded search with AND/OR, recent-paper browsing, seed-based related papers, cursor pagination

  <action>
    **db/queries.py**: Build SQLAlchemy query constructors:

    1. `build_search_query(query_text, title, author, category, date_from, date_to, time_basis, cursor, page_size) -> Select`:
       - Full-text search: If `query_text` provided, use `func.websearch_to_tsquery('english', query_text)` with `Paper.search_vector.op('@@')(tsquery)`. Order by `func.ts_rank_cd(Paper.search_vector, tsquery).desc()` then `Paper.arxiv_id`.
       - Title-only search: If `title` provided, create separate `plainto_tsquery('english', title)` against `to_tsvector('english', Paper.title)`.
       - Author search: If `author` provided, use `plainto_tsquery('simple', author)` against `to_tsvector('simple', Paper.authors_text)`. Use 'simple' config for names (not 'english' stemmer).
       - Category filter: If `category`, use `Paper.category_list.any(category)`.
       - Date range: Select column based on `time_basis` param ("submitted" -> submitted_date, "updated" -> updated_date, "announced" -> announced_date). Apply >= date_from, <= date_to as WHERE clauses.
       - Keyset cursor: If cursor provided, apply `WHERE (sort_col, arxiv_id) > (cursor.sort_value, cursor.paper_id)` for relevance-ordered results, or `WHERE (sort_col, arxiv_id) < (...)` for date-descending.
       - Limit: `page_size + 1` to detect has_next.
       - When no query_text (filter-only search), order by announced_date desc, then arxiv_id desc.

    2. `build_browse_query(category, time_basis, days, cursor, page_size) -> Select`: For recent paper browsing. Per research Example 4:
       - Compute cutoff date from `days` parameter
       - Select date column based on time_basis
       - Filter `WHERE date_col >= cutoff`
       - Filter category if provided via `Paper.category_list.any(category)`
       - Order by date_col DESC, arxiv_id DESC
       - Apply cursor and limit

    3. `build_related_query(seed_paper: Paper, page_size) -> Select`: For seed-based related papers. Per research Example 5 (Claude's discretion on similarity method -- use tsvector approach):
       - Extract seed paper's title + first 500 chars of abstract
       - Create `websearch_to_tsquery('english', combined_text)` as seed query
       - Rank corpus by `ts_rank_cd(search_vector, seed_query)` descending
       - Exclude the seed paper itself (`WHERE arxiv_id != seed_arxiv_id`)
       - Only match papers where `search_vector @@ seed_query`
       - Limit to page_size

    **search/pagination.py**: Implement keyset cursor application:
    - `apply_cursor(stmt, cursor, sort_column, descending=True) -> Select`: Adds the WHERE clause for keyset pagination using `(sort_column, Paper.arxiv_id)` compound comparison.
    - `build_page_info(items, page_size, sort_column_name) -> tuple[list, PageInfo]`: Takes page_size+1 items, removes extra, builds PageInfo with has_next and next_cursor encoded from last item.

    **search/ranking.py**: Result shaping:
    - `shape_search_results(rows, score_column=None) -> list[SearchResult]`: Convert SQLAlchemy Row objects to SearchResult Pydantic models, truncating abstract to abstract_snippet (300 chars).
    - Weight configuration: title=A, authors=B, abstract=C (matching the tsvector trigger weights).
