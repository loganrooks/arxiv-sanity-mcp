  <action>
    1. Create src/arxiv_mcp/interest/suggestions.py with:

       a. SuggestionCandidate dataclass: signal_type (str), signal_value (str), reason (str), score (float -- how strongly suggested, e.g., author frequency count)

       b. SuggestionService class:
          - __init__(session_factory, settings, profile_service: ProfileService)
          - async generate_suggestions(profile_slug) -> list[SuggestionCandidate]:
            Load profile to get existing signal_values (active + dismissed) to exclude.
            Call _suggest_seed_papers, _suggest_queries, _suggest_authors in sequence.
            Return combined candidates sorted by score descending.

          - async _suggest_seed_papers(session, existing_values, dismissed_values) -> list[SuggestionCandidate]:
            Query papers with TriageState in ("shortlisted", "cite-later").
            Exclude papers already in profile as seed_paper or negative_example signals.
            Return as SuggestionCandidate with reason "Triaged as {state}".

          - async _suggest_queries(session, existing_values, dismissed_values) -> list[SuggestionCandidate]:
            Query SavedQuery where run_count >= 3.
            Exclude queries already in profile as saved_query signals.
            Return with reason "Run {run_count} times".

          - async _suggest_authors(session, existing_values, dismissed_values) -> list[SuggestionCandidate]:
            Query papers with TriageState in ("shortlisted", "cite-later").
            Parse authors_text, normalize names, count frequency.
            Filter to authors appearing 3+ times not already in profile.
            Return with reason "Appears in {count} shortlisted/cite-later papers".
            Use normalize_author from interest/signals.py and parse_authors.

          - async add_suggestions_to_profile(profile_slug, candidates: list[SuggestionCandidate]) -> list[SignalInfo]:
            For each candidate, call profile_service.add_signal with source="suggestion", status="pending", reason=candidate.reason.
            Return list of created SignalInfo.

          - async confirm_suggestion(profile_slug, signal_type, signal_value) -> SignalInfo:
            Find the InterestSignal with status="pending" matching (profile, type, value).
            Update status to "active". Return updated SignalInfo.

          - async dismiss_suggestion(profile_slug, signal_type, signal_value) -> None:
            Find the InterestSignal with status="pending" matching (profile, type, value).
            Update status to "dismissed".

          - async confirm_suggestions_bulk(profile_slug, items: list[tuple[str, str]]) -> int:
            Confirm multiple suggestions. Return count confirmed.

    2. Write tests in tests/test_interest/test_suggestions.py:

       - TestSuggestAuthors: author with 3+ shortlisted papers suggested, reason includes count
       - TestSuggestQueries: high run_count queries suggested
       - TestConfirmDismiss: confirm changes pending to active, dismiss changes to dismissed
       - TestBulkConfirm: multiple suggestions confirmed at once
       - TestNoDuplicateSuggestions: re-running generate after dismiss doesn't re-suggest
       - TestEmptyWorkflow: no triage/queries returns empty suggestions
