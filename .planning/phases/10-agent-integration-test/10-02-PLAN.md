  <name>Task 1: Execute 5 E2E agent research session flows</name>
  <files>.planning/phases/10-agent-integration-test/10-SESSION-LOG.md</files>
  <action>
Human conducts a real research session in a FRESH Claude Code session with the MCP server active. The automated setup is complete (Plan 01). The human must pose natural language research questions and observe how the agent discovers and uses MCP tools.

**Flow 1 -- Literature Review (search -> triage -> collect):**
Prompt something like: "I'm interested in recent work on consciousness and phenomenology. Can you find relevant papers, help me identify the most important ones, and organize them into a collection?"
- Observe: Does the agent use search_papers? Does it triage papers? Does it create/use a collection?

**Flow 2 -- Interest-Driven Discovery (profile-ranked search):**
Prompt: "I have an interest profile called 'arxiv-scan-tensions'. Can you use it to find papers that match my research interests and explain why each paper scored well?"
- Observe: Does the agent use search_papers with profile_slug? Does it show ranking explanations?

**Flow 3 -- Watch/Delta Monitoring:**
Prompt: "Set up monitoring for new papers about 'philosophy of mind' so I can check for updates later."
- Observe: Does the agent use create_watch? Can it check for deltas?

**Flow 4 -- Content Access:**
Pick a paper from the results and prompt: "For that paper [cite the arxiv_id], can you get me the full content, not just the abstract?"
- Observe: Does the agent use get_content_variant? How does it handle rights/availability?

**Flow 5 -- Prompt-Guided:**
Use the literature_review_session prompt directly (or type something that triggers prompt usage).
- Observe: Does the agent discover and use MCP prompts?

**During the session, note EVERY friction point:**
- Wrong tool selected
- Confusing parameter names
- Unhelpful error messages
- Missing information in responses
- Unexpected behavior
- Tool descriptions that misled the agent
- Workflows that required workarounds

IMPORTANT: Do NOT tell the agent which tools to use. The whole point is to observe whether tool descriptions are clear enough for natural discovery.
