<context>

The user (Logan, philosophy PhD) held this spike program to extremely high epistemic standards. Five signals were logged for methodological rigor failures — each caught by the user, not self-identified:

1. Premature closure — declaring spikes complete before all protocol items checked
2. Gaps not proactively identified — B1 findings not propagated to C1 design
3. Measuring wrong thing — coverage against broken proxy instead of quality
4. Untested hypotheses as findings — mixture-of-experts recommended without testing null
5. Circular evaluation bias — MiniLM tested on MiniLM-defined clusters

Key epistemic lessons:
- Before presenting any comparison: "Is the evaluation biased toward any contestant by construction?"
- Before any metric: "What does this measure, and what does it NOT measure?"
- Before any conclusion: "What would change my mind?"
- "Quality" must be defined with explicit valuation framework before measuring
- Delegate qualitative review to agents rather than relying only on quantitative metrics
- Findings can be qualified. Open questions can be explicit. Finalizing ≠ resolving everything.

The user proposed: recommendation strategies as selectable profiles, tagged by relatedness type, combinable per project. This maps to existing InterestProfile + RankingExplanation architecture and ADR-0001.
