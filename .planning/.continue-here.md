## Spike program purpose reframed
Spikes characterize the trade-off space for different user situations. Output is a map of options with honest trade-off descriptions, not a verdict about which model wins. Different users (local-first vs quality-maximizing, GPU vs CPU-only, CS/ML vs interdisciplinary) will choose different configurations from the same characterization.

1. Spikes are not competitions — they characterize options for users with different situations
2. Representation is not function — knowing models produce different rankings says nothing about which rankings are better for researchers
