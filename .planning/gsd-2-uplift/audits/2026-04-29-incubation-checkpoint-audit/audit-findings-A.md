## §2. Findings

### F-IC-A1

**Class.** C

**Lens.** D5a leak / disposition-quality / negative-space

**Axis.** §7.4 six-context anchoring, with downstream effect on §7.3.a first-target-shape and §7.5 side-probes

**Finding.** INCUBATION-CHECKPOINT.md §7.4 disposes: "**Primary anchoring**: **A (solo-research-tool-over-years) and F (transition-as-stance / anticipatory-scaling) primary**" and "**plural-context-anchoring** per INITIATIVE.md §3.3 + framing-widening §3.3" (`INCUBATION-CHECKPOINT.md:400-407`). The reasoning says substrate-shape anchoring is plural-context with A+F primary partly because "arxiv-sanity-mcp's first-wave evidence is what we have" (`INCUBATION-CHECKPOINT.md:408-410`).

Diagnosis: the disposition keeps the words "plural-context-anchoring" but operationally re-centers the available test case. That is not fully licensed by the required grounding. `RELATIONSHIP-TO-PARENT.md` says arxiv-sanity-mcp is "a spike-intensive test case" and explicitly warns that it is "*a* spike-intensive test case, not *the* test case" (`RELATIONSHIP-TO-PARENT.md:31-40`). `framing-widening.md` says if the initiative is for gsd-2 users more broadly, arxiv-sanity-mcp is "one user-context among many" and project-anchoring should be plural (`framing-widening.md:196-206`). `INITIATIVE.md §3.3` also preserves onboarding situations beyond the current project, including raw Claude Code projects, organization adoption, preview mode, and selective uplift (`INITIATIVE.md:92-108`).

**Reasoning.** The checkpoint already flags this as the highest D5a risk in §7.8 (`INCUBATION-CHECKPOINT.md:461-470`), but the disposition's reasoning still uses the arxiv-sanity evidence availability as a positive reason for A+F-primary substrate anchoring. Availability is a good reason to choose a diagnostic first case; it is weaker evidence for substrate-shape primary anchoring. The missing distinction is between "Phase D diagnostic anchor" and "substrate-shape anchoring." If Logan intends the first, the checkpoint should say so; if Logan intends the second, the plural-context claim needs either additional evidence or a narrower status label.

**Confidence.** High

**Non-binding disposition signal.** Re-disposition if A+F-primary is meant as substrate-shape anchoring. Addendum if Logan means "A+F diagnostic first case under plural-context substrate discipline."

### F-IC-A2

**Class.** C

**Lens.** Clarification-quality / disposition-quality

**Axis.** Cross-axis: §7.1 R4 disposition-timing + §7.3.a first-target-shape + §7.5 P5

**Finding.** §7.1 chooses "(b) evaluate-whether-to-shift" and states that the "first-target generates evidence ABOUT the shift" (`INCUBATION-CHECKPOINT.md:337-347`). §7.3.a then chooses "Context A/F long-arc decision-trace skill/workflow" as a "R2-shaped first-target on the skill subsystem" (`INCUBATION-CHECKPOINT.md:364-374`), while §7.3.c says the chosen decomposition is "Skills subsystem primarily" and only "possibly composing with workflow templates" (`INCUBATION-CHECKPOINT.md:390-395`).

Diagnosis: a primarily R2 skill/workflow target does not automatically generate evidence about whether incubation should shift to an R2+R4 operating frame. The R4 evidence route is present only indirectly: P5 pre-D parallel (§7.5), possible workflow-template composition (§7.3.c), or later Phase E re-disposition. That leaves an internal incoherence between the §7.1 evidence claim and the §7.3 dispatch shape.

**Reasoning.** `SYNTHESIS-COMPARISON.md §2.1` defines the §7.1 choice as a deliberation-shape difference: under (a), incubation operates under R2+R4; under (b), incubation asks whether to shift and on what evidence (`SYNTHESIS-COMPARISON.md:150-163`). `SYNTHESIS-COMPARISON.md §5.2` says first-target selection should identify which R2 subsystem carries the cheapest viable first target and whether incubation operates under the shifted mix (`SYNTHESIS-COMPARISON.md:298-312`). The checkpoint's selected target tests R2 skill viability well, but it needs an explicit R4 observable or comparator to test the shift. Otherwise Phase D may produce good R2 evidence while leaving §7.1's main question under-evidenced.

**Confidence.** High

**Non-binding disposition signal.** Re-disposition if §7.1's evidence-about-shift claim is load-bearing as written. Addendum if Phase D dispatch can require an explicit R4 contrast, such as a headless/orchestration comparator, effective-state-emission integration, or a declared "R4 not tested by this first target" limit.

### F-IC-A3

**Class.** B

**Lens.** Clarification-quality / negative-space

**Axis.** §7.5 side-probe pre-vs-post, especially P5 pre-D parallel

**Finding.** §7.5 disposes P5 effective-state-emission as "**pre-Phase-D parallel**" because it is cheap and useful diagnostic context (`INCUBATION-CHECKPOINT.md:423-439`). It does not specify what happens if P5 reveals docs-vs-source drift severe enough to invalidate the decision-trace skill/workflow assumptions before Phase D starts.

Diagnosis: P5 is treated as helpful context, not as a gate with a failure branch. That is under-specified relative to the probe's stated purpose.

**Reasoning.** `SYNTHESIS-COMPARISON.md §5.4` defines P5's trigger as the docs-vs-source-drift class making any docs-trusting R-strategy risky, with a source/run check of `prefs`, `headless query`, or MCP tools resolving complete effective settings (`SYNTHESIS-COMPARISON.md:332-347`). `META-SYNTHESIS.md` preserves the same boundary: effective-state, runtime-surface, and source-verification details are addenda incubation must carry forward, not assumptions to smooth over (`META-SYNTHESIS.md:36-45`, `META-SYNTHESIS.md:69-86`). Because P5 is pre-D, a severe result should have a pre-D disposition path: pause first-target dispatch, re-scope the target, or proceed with an explicit invalidation caveat.

**Confidence.** Medium-high

**Non-binding disposition signal.** Addendum. Define P5 outcome branches before Phase D dispatch.

### F-IC-A4

**Class.** B

**Lens.** Negative-space / disposition-quality

**Axis.** §7.5 P2 temporal-stability disposition and §7.8 audit-priority list

**Finding.** §7.5 disposes P2 temporal-stability "post-Phase-D" because skill subsystem stability matters downstream but is "not for first-target spike" (`INCUBATION-CHECKPOINT.md:423-440`). §7.8 does not include P2 post-D as a prioritized challenge surface, despite §7.3 acknowledging that skill subsystem temporal stability is untested (`INCUBATION-CHECKPOINT.md:390-397`, `INCUBATION-CHECKPOINT.md:461-472`).

Diagnosis: this under-prioritizes a directly relevant probe. The selected first target is R2-shaped on the skill subsystem, and the upstream comparison says P2 is triggered by any R2 work that targets a specific subsystem and requires stability evidence beyond the shallow window.

**Reasoning.** `SYNTHESIS-COMPARISON.md §5.4` defines P2 as a temporal-stability probe across extension subsystems and says its trigger is "any R2 work targeting a specific subsystem" requiring stability evidence (`SYNTHESIS-COMPARISON.md:332-347`). `framing-widening.md §9` likewise says the temporal-stability probe is "triggered before any R2 work that depends on specific surfaces" (`framing-widening.md:360-368`). The checkpoint can still choose post-D for momentum, but the rationale should explain why the first R2 skill/workflow spike does not depend on the skill surface's stability, or should make P2 a small pre-D spot-check rather than a broad post-D probe.

**Confidence.** High

**Non-binding disposition signal.** Addendum or narrow revision. At minimum, §7.8 should include P2 post-D as a challenge surface; stronger version: add a bounded pre-D P2 spot-check for the skill subsystem only.

## §3. Negative-space observations

1. **§7.8 converges on the right top risks but misses two adjacent surfaces.** It correctly elevates §7.4, §7.1, §7.3.a, P5, §7.6, and §7.7. It misses P2 temporal-stability as a near-term challenge surface for the selected R2 skill subsystem, and it misses user-side adoption-pattern evidence as the probe most directly tied to plural-context substrate anchoring.

2. **First-target-first remains under-explored as evidence generation rather than ordering preference.** The checkpoint treats first-target-first mainly as a competing deliberation order (§7.6). The stronger negative-space question is what first-target-first would have surfaced about anchoring: e.g., an effective-state-emission probe could have tested R4/runtime visibility before A+F decision-trace work made A+F feel inevitable.

3. **"Plural-context" is asserted more than operationally represented.** B is adjacent, E secondary, C/D out, but the Phase D target and probes mostly preserve A/F. That may be principled as a diagnostic first case, but it should not be mistaken for broad substrate-shape coverage.

4. **Frame revision is available later but not active now.** This is principled if "active" means "must revise before Phase D"; it is accidental if "none active" means "no current stress." There are current stressors: R4/R2 boundary fuzziness, six-context over-fragmentation, and strategy/act redundancy are all already named in framing-widening §10.
