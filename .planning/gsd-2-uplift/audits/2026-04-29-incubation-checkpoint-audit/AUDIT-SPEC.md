# AUDIT-SPEC.md — Phase C Incubation-Checkpoint Audit

## §0. Purpose

Phase C of the trajectory plan (`.planning/gsd-2-uplift/trajectory/cheerful-forging-galaxy.md`) executes incubation-checkpoint adjudication on SYNTHESIS-COMPARISON.md §2.1 + §5 axes. Logan disposed the checkpoint per axis 2026-04-29 by accepting Claude's xhigh-effort rendered-transparent position with reasoning, steel-man counters, sensitivity tables, and audit-priority risk disclosure. This audit pressure-tests the disposed checkpoint before it becomes load-bearing for Phase D first-target dispatch.

**Why this audit exists.** The checkpoint dispositions carry D5a in-session-collaboration risk (Claude drafted the surfacing artifact + the rendered-transparent proposal; Logan accepted-as-proposed). Claude's rendered transparency surfaces own-most-likely-wrong-where flagging at INCUBATION-CHECKPOINT.md §7.8, but rendered transparency is NOT a substitute for independent audit — auditors should treat §7.8 as a *prioritized challenge surface*, not a self-audit that has already done the work. Where the audit converges on §7.8, the convergence is calibration-grade evidence about transparency-coverage; where the audit surfaces additional risks beyond §7.8, that's the value-add of independent reading.

**Why paired.** Per premise-bleed audit precedent (DECISION-SPACE.md §1.17 lessons-distilled): cross-vendor + same-vendor-xhigh pairing produces complementary findings rather than redundant ones. Cross-vendor catches vocabulary-import / substantive divergence; same-vendor catches integration-grammar-as-fact at meta-level (the §7.1 reading-frame residual the comparison documented). Both vendor-positions are needed for premise-bleed and framing-leak because either alone is structurally incomplete.

**Why xhigh both.** Per trajectory plan §2.4 row C: Logan dispositions on §5 axes inherit Logan-framing inheritance; decision-stake shapes Phase D first-target which shapes substrate-shape evidence which shapes extraction-readiness; cross-cutting integrates synthesis + comparison + framing-widening + audit-arc outcomes. xhigh warranted on both auditors.

## §1. Lens for the audit

**Primary lens — disposition-quality challenge.** The audit's job is to challenge the disposed checkpoint, not echo it. Disposition-quality failures would look like:
- Disposition not actually licensed by the surfaced evidence (e.g., (β) metaquestion presupposes A+F-primary anchoring; if anchoring evidence is weaker than disposition implies, (β) over-states).
- Reasoning trace conflating projection with evidence (e.g., test-case-anchoring projected onto substrate-shape-anchoring without first-wave evidence licensing).
- Steel-man counter under-developed (auditor's read of the same axis surfaces stronger counter than Claude rendered).
- Sensitivity table missing salient parameters (auditor identifies parameters that would shift disposition that Claude didn't surface).
- Audit-priority risk flagging mis-prioritized (auditor reads different axes as higher-risk than §7.8 ordered).

**Secondary lens — D5a in-session-collaboration leak detection.** The dispositions are Claude-drafted in in-session-collaboration. D5a leaks would look like:
- Substrate-shape projection from test-case-anchoring without explicit framing-discipline (RELATIONSHIP-TO-PARENT.md → INCUBATION-CHECKPOINT.md inheritance flagged in §7.8 (i); audit verifies it's only flagged not actually leaking).
- Conservative-default-when-uncertain (e.g., (b) over (a) §2.1 — flagged in §7.8 (ii); audit verifies whether (b) is principled disposition-discipline-honoring or default-conservative).
- Premature commitments in dispositions that should be conditional (e.g., first-target selection presupposing dispositions that are themselves under audit).

**Tertiary lens — framing-leak detection (§7.1 reading-frame residual at meta-level).** The dispositions cite framing-widening + comparison + INITIATIVE.md vocabulary; framing-leak would look like:
- Treating R1-R5 / six-context / four-act vocabulary as observed-fact in gsd-2 rather than as Logan-imported inputs (the §7.1 residual the premise-bleed audit caught at synthesis stage now potentially recurring at disposition stage).
- Frame-revision-none-active disposition (§7.7) treated as neutral when it might be active-non-revision (where gradient phenomena are forced into categorical answers).
- Integration grammar treated as same-kind-as observed-evidence at disposition stage when at synthesis stage it was explicitly framed as inputs-not-facts.

**Quaternary lens — clarification quality / Phase D readiness.** Does the disposed checkpoint actually clarify what Phase D should dispatch? Clarification-quality failures would look like:
- Operational ambiguity at first-target dispatch (e.g., "decision-trace skill/workflow" doesn't specify enough for Phase D dispatch).
- Internal incoherence between dispositions (e.g., §7.1 (b) but §7.3.a first-target chosen as if (a) were disposed).
- Missing the load-bearing case (e.g., what happens at Phase D if effective-state-emission probe (P5 pre-D parallel) reveals docs-vs-source-drift severe enough to invalidate decision-trace assumptions?).

**Negative-space lens (required, per DECISION-SPACE.md §1.17 audit-methodology rule 2).** What is the disposed checkpoint *excluding* from consideration? Audit-priority risk surfaces (per §7.8) name some exclusions; auditor identifies others. Specifically check: (i) are there axes that should have been disposed but weren't?; (ii) are there options within axes that were dismissed without explicit reasoning?; (iii) does the deliberation-order anchoring-first hide what first-target-first would have surfaced?

## §2. Methodology

Per AUDIT-SPEC.md §6.1 universal audit-output structure (established by premise-bleed audit-arc precedent):

1. **Read the artifact under audit** (`INCUBATION-CHECKPOINT.md` full §0-§8, including §7 disposition record + §7.8 audit-priority list).
2. **Read the upstream-grounding** (SYNTHESIS-COMPARISON.md §0-§7 with audit addendum; DECISION-SPACE.md §1.18 entry; INITIATIVE.md §3.1 (β)-note; framing-widening §1-§3 + §9; trajectory plan §1.3 + §2.4 row C).
3. **Apply the four lenses** (disposition-quality / D5a leak / framing-leak / clarification-quality) systematically per axis (§7.1-§7.7).
4. **Apply negative-space lens** at the disposed checkpoint as a whole and at each axis.
5. **Calibrate confidence** per finding (high / medium / low) and class per finding (Class A: addendum-shape; Class B: revision-shape; Class C: re-architect-shape).
6. **Produce non-binding disposition signal** per finding + overall.

**Independent mode (both auditors).** Per AUDIT-SPEC.md §3.4 manual-discretion pattern (premise-bleed precedent + Phase B precedent): each auditor reads ONLY the artifacts in §3 below. Do NOT read each other's findings before completing own audit; this preserves independence-of-finding and avoids cross-auditor-anchoring. Do NOT read prior audit folders' findings (premise-bleed FINDINGS / FINDINGS-STEP2 / DIFFERENTIAL / DISPOSITION; trajectory-plan PLAN-AUDIT / DISPOSITION; relationship-to-parent audit-findings-A/B / DISPOSITION) before completing own audit.

**Cross-vendor + same-vendor as paired Step-1 + Step-2 (not Step-1-then-Step-2-conditional).** Per premise-bleed audit's lessons-distilled (DECISION-SPACE.md §1.17): paired Step-1 + Step-2 produces complementary findings, not redundant. This audit invokes both as Step-1 and Step-2 in parallel (independent dispatch); main-thread differential analysis post-hoc reconciles per premise-bleed precedent.

## §4. Output structure

Write findings to `audit-findings.md` (single auditor) OR `audit-findings-A.md` + `audit-findings-B.md` (paired auditors; A=cross-vendor codex; B=same-vendor adversarial-auditor) in this audit folder. Naming-convention rationale per Phase B precedent: `audit-findings*.md` bypasses Claude Code 2.1.123's `tengu_sub_nomdrep_q7k` regex.

**§0. Summary** — auditor identity + reasoning-level + class breakdown + headline + non-binding overall disposition signal.

**§1. Methodology applied** — what was read; what was excluded; mode (independent); time budget per artifact.

**§2. Findings** — per-finding entries with this structure:
  - **ID** (e.g., F-IC-1).
  - **Class** (A / B / C per AUDIT-SPEC.md §6.1 stratification).
  - **Lens** (disposition-quality / D5a leak / framing-leak / clarification-quality / negative-space).
  - **Axis** (which §7.X disposition the finding bears on; or "cross-axis" for findings that span multiple).
  - **Finding** (verbatim text from artifact + concise diagnosis).
  - **Reasoning** (why this is a finding; what evidence supports the call).
  - **Confidence** (high / medium / low).
  - **Non-binding disposition signal** (per-finding: addendum / revision / re-disposition / no-action).

**§3. Negative-space observations** — what the disposed checkpoint excludes; whether principled or accidental; specifically whether §7.8 audit-priority list missed surfaces the auditor identifies.

**§4. Calibration** — auditor's confidence in own findings; what would shift the call; coverage assessment relative to §7.8 prioritized challenge surfaces (converged-with / extended-beyond / mis-aligned-with).

**§5. Non-binding overall disposition signal** — recommendation per AUDIT-SPEC.md §8 disposition pathway: commit-as-is / commit-with-addendum / revise-before-commit. Per-option justification (why Logan might pick each given the findings); auditor does not pick one.
