<objective>
Build the composable ranking pipeline with five signal scorers, ProfileRankingService that wraps WorkflowSearchService, and the ProfileSearchResult/ProfileSearchResponse types.

Purpose: This is the core differentiator -- every search result gets an inspectable, structured ranking explanation showing exactly why it surfaced. The ranking pipeline is composable so Phase 4 enrichment signals (citations, FWCI) can plug in without restructuring.
Output: RankingPipeline, ProfileRankingService, ProfileSearchResult, RankerSnapshot, and tests.
</objective>

       a. SignalType(StrEnum): QUERY_MATCH, SEED_RELATION, CATEGORY_OVERLAP, INTEREST_PROFILE_MATCH, RECENCY

       b. SignalScore dataclass: signal_type, raw_score, normalized_score, weight, weighted_score, explanation

       c. RankingExplanation dataclass: composite_score, signal_breakdown (list[SignalScore]), ranker_version

       e. RankerSnapshot dataclass: profile_slug (str|None), ranker_version, weights (dict), signal_types_applied (list[str]), seed_paper_count, followed_author_count, negative_example_count, saved_query_count, negative_weight, captured_at (datetime)

          - score_query_match(query_rank, all_ranks) -> SignalScore: min-max normalization, zero-division guard
          - score_category_overlap(paper, seed_categories) -> SignalScore: Jaccard similarity on category_list
          - score_recency(paper, max_date, decay_days=90) -> SignalScore: linear decay
          - score_seed_relation(paper, seed_papers, seed_categories, followed_authors) -> SignalScore: composite of category overlap with seeds + author match (proxy for lexical similarity without N queries). Exclude papers whose arxiv_id is in seed set.
          - score_profile_match(paper, profile_context) -> SignalScore: normalized aggregation of seed_relation + category_overlap + author match sub-signals (the composite "how well does this match your profile" number). Avoids double-counting by using a single composite formula.

       g. apply_negative_demotion(scores, paper, profile_context) -> None (mutates scores list): if paper categories overlap with negative_categories or paper arxiv_id is a negative example, multiply all weighted_scores by (1.0 - negative_weight). This is soft demotion -- never removes scores entirely.

       h. RankingPipeline class:
          - RANKER_VERSION = "0.3.0"
          - DEFAULT_WEIGHTS = {QUERY_MATCH: 0.35, SEED_RELATION: 0.25, CATEGORY_OVERLAP: 0.15, INTEREST_PROFILE_MATCH: 0.15, RECENCY: 0.10}
          - __init__(weights=None): merge with DEFAULT_WEIGHTS
          - score_paper(paper, query_rank=None, all_ranks=None, profile_context=None) -> RankingExplanation: calls applicable scorers, applies negative demotion if needed, sums weighted_scores for composite
          - capture_snapshot(profile_context=None) -> RankerSnapshot: captures current pipeline config

       - __init__(session_factory, settings, workflow_search_service: WorkflowSearchService)
       - async search_papers(*, profile_slug=None, **kwargs) -> ProfileSearchResponse:
         If no profile_slug: delegate to workflow_search, wrap results as ProfileSearchResult (no ranking_explanation), return ProfileSearchResponse(results=..., ranker_snapshot=None).
         If profile_slug provided:
           a. Override page_size to page_size * 3 in kwargs for over-fetch (Pitfall 2 mitigation)
           b. Call workflow_search.search_papers(**modified_kwargs)
           c. Load ProfileContext via _load_profile_context(profile_slug)
           d. Build RankingPipeline with profile weights
           e. Collect all_ranks from result items (for min-max normalization)
           f. Score each paper via pipeline.score_paper
           g. Build ProfileSearchResult for each (preserving triage_state, collection_slugs from workflow result)
           h. Sort by composite_score descending
           i. Trim to original page_size
           j. Capture ranker_snapshot via pipeline.capture_snapshot(profile_context)
           k. Return ProfileSearchResponse with results + snapshot
       - async browse_recent(*, profile_slug=None, **kwargs) -> ProfileSearchResponse:
         Same pattern as search_papers.
       - async _load_profile_context(profile_slug) -> ProfileContext:
         Load profile from DB with signals.
         Batch-load seed paper metadata: SELECT * FROM papers WHERE arxiv_id IN (:seed_ids) (Pitfall 4 mitigation).
         Batch-load negative paper metadata similarly.
         Build ProfileContext with seed_categories as union of all seed paper category_lists, followed_authors list, query_slugs, weights from profile (or DEFAULT_WEIGHTS), negative_weight.
       - _wrap_without_ranking(result) -> ProfileSearchResponse:
         Convert WorkflowSearchResult items to ProfileSearchResult items (ranking_explanation=None), wrap in ProfileSearchResponse(ranker_snapshot=None).
