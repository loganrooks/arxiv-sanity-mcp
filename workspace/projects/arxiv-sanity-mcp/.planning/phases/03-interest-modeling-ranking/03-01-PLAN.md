    1. Create src/arxiv_mcp/interest/profiles.py with ProfileService class:
       - __init__(session_factory, settings) following established service DI pattern
       - create_profile(name, negative_weight=None, source="manual") -> ProfileSummary:
         Uses slugify() from workflow.util. Checks soft limit, warns via structlog if exceeded. Sets created_at/updated_at to now.
       - list_profiles(include_archived=False) -> list[ProfileSummary]:
         Query InterestProfile, optionally filter is_archived=False. Add signal_count as subquery count.
       - get_profile(slug) -> ProfileDetail:
         Eagerly load signals. Compute signal_counts_by_type (dict of type -> count of active signals) and signal_counts_by_source (dict of source -> count). Return ProfileDetail.
       - rename_profile(slug, new_name) -> ProfileSummary:
         Update name, regenerate slug, update updated_at. Raise if new slug conflicts.
       - archive_profile(slug) / unarchive_profile(slug) -> ProfileSummary:
         Set is_archived flag.
       - delete_profile(slug) -> None:
         Delete profile (cascade removes signals).
       - add_signal(slug, signal_type, signal_value, source="manual", reason=None) -> SignalInfo:
         Core method. Validates signal_type via signals.validate_signal. For seed_paper/negative_example: verify Paper exists (FK). For saved_query: verify SavedQuery exists (warn via log if missing but still allow — per resilience decision, warn-not-error). For followed_author: normalize via normalize_author. Check for existing (profile_id, signal_type, signal_value) — raise if exists. Create InterestSignal with status="active" (or "pending" if source="suggestion").
       - remove_signal(slug, signal_type, signal_value) -> None:
         Find and delete the signal row. Raise if not found.
       - Convenience wrappers: add_seed_paper, remove_seed_paper, add_saved_query, remove_saved_query, add_followed_author (follow), remove_followed_author (unfollow), add_negative_example, remove_negative_example — each calls add_signal/remove_signal with the correct signal_type.
