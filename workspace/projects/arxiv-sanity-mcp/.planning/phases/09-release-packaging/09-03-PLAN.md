<task type="auto">
  <name>Task 1: Create CI workflow and GitHub repository</name>
  <files>.github/workflows/ci.yml</files>
  <action>
1. Create `.github/workflows/` directory.

2. Create `.github/workflows/ci.yml` with the following configuration:
   - Name: "CI"
   - Triggers: push to main, pull_request to main
   - Single job "test" on ubuntu-latest
   - PostgreSQL 16 service container:
     ```yaml
     services:
       postgres:
         image: postgres:16
         env:
           POSTGRES_USER: postgres
           POSTGRES_PASSWORD: postgres
           POSTGRES_DB: postgres
         options: >-
           --health-cmd pg_isready
           --health-interval 10s
           --health-timeout 5s
           --health-retries 5
         ports:
           - 5432:5432
     ```
   - Steps (in order):
     a. `actions/checkout@v5`
     b. `actions/setup-python@v6` with python-version '3.13' and cache 'pip'
     c. Install dependencies: `pip install -e . && pip install pytest pytest-asyncio pytest-cov pytest-timeout respx ruff`
        (Use explicit dev dep install because pyproject.toml uses PEP 735 dependency-groups which pip may not support via --group flag)
     d. Create test database and user:
        ```bash
        PGPASSWORD=postgres psql -h localhost -U postgres <<SQL
        CREATE USER arxiv_mcp WITH PASSWORD 'arxiv_mcp_dev';
        CREATE DATABASE arxiv_mcp OWNER arxiv_mcp;
        CREATE DATABASE arxiv_mcp_test OWNER arxiv_mcp;
        GRANT ALL PRIVILEGES ON DATABASE arxiv_mcp TO arxiv_mcp;
        GRANT ALL PRIVILEGES ON DATABASE arxiv_mcp_test TO arxiv_mcp;
        SQL
        ```
     e. Run migrations: `alembic upgrade head`
        with env: DATABASE_URL=postgresql+asyncpg://arxiv_mcp:arxiv_mcp_dev@localhost:5432/arxiv_mcp
     f. Run tests: `python -m pytest tests/ --timeout=30`
        with env: DATABASE_URL and TEST_DATABASE_URL set appropriately
     g. Lint: `ruff check src/ tests/`

3. Create the GitHub repository and push:
   ```bash
   gh repo create loganrooks/arxiv-sanity-mcp --public --source=. --remote=origin --push
   ```
   If this fails due to SSH key issues, fall back to:
   ```bash
   gh repo create loganrooks/arxiv-sanity-mcp --public
   git remote add origin https://github.com/loganrooks/arxiv-sanity-mcp.git
   git push -u origin main
   ```
   The gh CLI is authenticated (loganrooks, repo+workflow scopes). Git operations protocol is SSH per gh auth status.

4. After push, wait for CI to trigger. Check status:
   ```bash
   gh run list --workflow=ci.yml --limit=1
   ```
   If CI fails, diagnose and fix. Common issues:
   - Database setup: user/password mismatch (verify psql commands)
   - Alembic: asyncpg connection string format
   - Dependencies: missing dev packages in pip install step
   - Ruff: any remaining lint errors (should be zero after Plan 01)
  </action>
