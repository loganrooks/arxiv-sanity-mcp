This is NOT a pytest. This is a real workflow using actual MCP tool functions against the imported database. Call the tool functions directly (via Python async calls against the running services) to simulate what an MCP client would do:

3. **Collect phase:** Call add_to_collection to group shortlisted papers into a "validation-test" collection. Observe the workflow.

4. **Expand phase:** Call find_related_papers with 2-3 seed IDs from shortlisted papers. Check whether the 7 false-negative papers surface in related results (this is a key validation test from CONTEXT.md).

5. **Enrich phase:** Call enrich_paper for 3-5 papers. Note: this makes real OpenAlex API calls. If OpenAlex is unavailable, document the limitation.

6. **Profile phase:** Use the "arxiv-scan-tensions" profile created by the import. Call search_papers with profile context to observe ranking differences.
