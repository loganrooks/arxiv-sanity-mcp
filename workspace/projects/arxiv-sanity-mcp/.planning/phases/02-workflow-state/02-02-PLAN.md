```python
class Collection(Base):
    __tablename__ = "collections"
    id: Mapped[int]  # autoincrement PK
    slug: Mapped[str]  # unique
    name: Mapped[str]
    is_archived: Mapped[bool]  # default False
    created_at: Mapped[datetime]
    updated_at: Mapped[datetime]
    paper_associations: Mapped[list["CollectionPaper"]] = relationship(back_populates="collection")

class CollectionPaper(Base):
    __tablename__ = "collection_papers"
    collection_id: Mapped[int]  # FK collections.id, composite PK
    paper_id: Mapped[str]  # FK papers.arxiv_id, composite PK
    source: Mapped[str]  # "manual", "saved_query", "agent"
    added_at: Mapped[datetime]
    collection: Mapped["Collection"] = relationship(back_populates="paper_associations")
    paper: Mapped["Paper"] = relationship()

class TriageState(Base):
    __tablename__ = "triage_states"
    paper_id: Mapped[str]  # PK, FK papers.arxiv_id
    state: Mapped[str]  # CHECK constraint: shortlisted, dismissed, read, cite-later, archived
    updated_at: Mapped[datetime]

class TriageLog(Base):
    __tablename__ = "triage_log"
    id: Mapped[int]  # autoincrement PK
    paper_id: Mapped[str]  # FK papers.arxiv_id
    old_state: Mapped[str]
    new_state: Mapped[str]
    timestamp: Mapped[datetime]
    source: Mapped[str]  # default "manual"
    reason: Mapped[str | None]
```

    Create src/arxiv_mcp/workflow/triage.py with TriageService class:

    ```python
    class TriageService:
        VALID_STATES = {"unseen", "shortlisted", "dismissed", "read", "cite-later", "archived"}

        def __init__(self, session_factory, settings): ...

        async def mark_triage(self, paper_id: str, new_state: str,
                              source: str = "manual", reason: str | None = None) -> TriageStateResponse
        async def get_triage_state(self, paper_id: str) -> str
        async def list_by_state(self, state: str, page_size: int = 20,
                                cursor_token: str | None = None) -> PaginatedResponse[TriageStateResponse]
        async def batch_triage(self, paper_ids: list[str], new_state: str,
                               source: str = "manual", reason: str | None = None) -> BatchTriageResult
        async def batch_triage_by_query(self, query_params: dict, new_state: str,
                                         dry_run: bool = True, source: str = "batch",
                                         reason: str | None = None) -> BatchTriageResult | BatchTriagePreview
        async def get_triage_log(self, paper_id: str, limit: int = 50) -> list[TriageLogEntry]
    ```

    Key implementation details:
    - Absence-means-unseen pattern (per locked decision): No TriageState row = "unseen". Query with LEFT JOIN + CASE expression.
    - mark_triage to "unseen": DELETE the TriageState row (revert to absence). Still log the transition.
    - mark_triage to any other state: UPSERT (INSERT or UPDATE). Log the transition with old_state (get current state first).
    - Validate new_state against VALID_STATES before any DB operation. Raise ValueError for invalid states.
    - batch_triage (ID-based): Process all paper_ids in a single transaction. Track affected_count (state actually changed), skipped_count (already in target state), and errors (paper_id not found in papers table). Validate paper existence before triage.
    - **batch_triage_by_query (query-based)** per CONTEXT.md locked decision "Batch triage supports both explicit ID lists AND query-based bulk operations" and "Query-based batch triage: dry-run by default, requires --confirm to execute":
      - Accepts query_params dict with same keys as search (query_text, title, author, category, date_from, date_to, time_basis, triage_filter, collection_filter).
      - Uses Phase 1's build_search_query() to resolve matching paper IDs. Remove pagination limit to get all matching IDs.
      - When dry_run=True (default): Return BatchTriagePreview with matching_count (total papers matching), sample_ids (first 10 IDs as preview), and query_description (human-readable summary of the query params). Does NOT modify any state.
      - When dry_run=False: Execute the triage on all matching papers using batch_triage() internally. Process in chunks of 500 to avoid transaction size issues (per research pitfall 3). Return BatchTriageResult with affected_count, skipped_count, errors.
      - Source defaults to "batch" for query-based operations (distinguishable in audit log).
    - list_by_state("unseen"): Use LEFT JOIN with papers table, WHERE TriageState.state IS NULL. For other states: simple WHERE clause.
    - get_triage_log: ORDER BY timestamp ASC (chronological). Return TriageLogEntry Pydantic models.
    - All operations within async context manager from session_factory.

    Create tests/test_workflow/test_triage.py with integration tests. Tests should pre-populate papers in conftest, then exercise all triage operations including both batch modes.
